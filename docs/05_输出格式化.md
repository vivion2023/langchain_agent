## 2. 背景与概念

### 输出格式化的作用

将大模型的原始输出解析为结构化的数据格式（JSON、字典、对象等），便于处理，提高输出的可读性和一致性。


输出格式化在两端配合完成：

```
PromptTemplate + format_instructions  →  告诉模型「按什么结构输出」
StructuredOutputParser.parse()        →  把模型回复解析成 Python 字典
```

### 核心组件

| 组件 | 作用 |
|------|------|
| `ResponseSchema` | 定义每个字段的名称、描述、类型 |
| `StructuredOutputParser` | 生成格式说明，并解析模型输出 |
| `partial_variables` | 将 `format_instructions` 预填入模板，无需每次手动拼接 |

> LangChain 1.x 中旧版 `langchain.output_parsers` 已迁移至 `langchain_classic.output_parsers`（随 `langchain-community` 安装）。

## 4. 实验：StructuredOutputParser

### 核心思路

1. 用 `ResponseSchema` 声明要提取的字段（`name`、`age`）
2. `StructuredOutputParser` 自动生成 JSON 格式说明，注入提示词
3. 模型返回后，用 `output_parser.parse()` 得到字典

### 代码

对应 Notebook **04 StructuredOutputParser** 单元格：

```python
from langchain_classic.output_parsers import ResponseSchema, StructuredOutputParser
from langchain_core.prompts import PromptTemplate

# 定义输出结构
response_schemas = [
    ResponseSchema(name="name", description="人的姓名", type="string"),
    ResponseSchema(name="age", description="人的年龄", type="integer"),
]

# 创建输出解析器
output_parser = StructuredOutputParser.from_response_schemas(response_schemas)

# 创建提示模板，包含格式化指令
template = """你是一个信息提取助手，请从以下文本中提取姓名和年龄，并以 JSON 格式返回:
文本: {input_text}
{format_instructions}"""

prompt = PromptTemplate(
    template=template,
    partial_variables={"format_instructions": output_parser.get_format_instructions()},
)

# 填充输入
input_text = "张三今年25岁，来自北京。"
filled_prompt = prompt.format(input_text=input_text)

# 获取输出
resp = chat.invoke(filled_prompt)
parsed_output = output_parser.parse(resp.content)
print(parsed_output)
```

### 预期输出

```python
{'name': '张三', 'age': 25}
```

### 关键点

- `get_format_instructions()` 会生成带字段说明的 JSON 模板，通过 `partial_variables` 固定注入，调用时只需传 `input_text`
- `parse()` 负责从模型回复（可能含 Markdown 代码块）中提取 JSON 并转为字典
- `ResponseSchema` 的 `type` 用于生成格式说明，帮助模型理解字段类型

## 6. 常见错误与排查

| 报错 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: langchain.output_parsers` | LangChain 1.x 已移除旧路径 | 改为 `from langchain_classic.output_parsers import ...` |
| `OutputParserException` | 模型输出不符合 JSON 格式 | 检查提示词是否包含 `format_instructions`；可重试或换更强模型 |
| `NameError: chat is not defined` | 未先运行环境搭建单元格 | 先运行 Cell 01 |

## 7. 我的理解

- 提示词模板解决的是「输入侧结构化」；输出格式化解决的是「输出侧结构化」，两者常配合使用。
- `format_instructions` 相当于给模型一份「答题卡」，告诉它每个空填什么类型。
- `parse()` 把非结构化的自然语言回复收拢成程序可直接使用的字典，是 Agent 串联多步逻辑的前提。
